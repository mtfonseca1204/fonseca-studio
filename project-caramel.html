CONTEXT

Token launches still feel like developer tools

Wallet interactions and unfamiliar parameters create friction instead of excitement. The gap wasn’t interest, it was the interface. Built at BlockchainRio 2024 on LaChain, inspired by the pump.fun model.

First-time creators wanted to launch meme coins but stalled at wallet setup and opaque contract parameters. The product had to feel like a consumer app while keeping on-chain interactions visible enough to earn trust.

APPROACH

Hackathon speed with consumer UX principles

Hackathon cycle with developer Bruno Eleodo. Market analysis and quick validation confirmed users wanted to launch, but existing tools felt too complex.

One decision at a time. Show enough on-chain detail to build trust without overwhelming. Every screen had to earn its place under time pressure.

SOLUTION

Idea → create → launch → on-chain

Step-by-step flows, clear interface patterns, and transparent smart contract interactions that abstract complexity without hiding how the system works. Playful, internet-native branding structured enough to earn product trust.

DECISIONS

Three tradeoffs that shaped the product

Step-by-step over dashboard complexity

We considered exposing every on-chain parameter at once. First-time users stalled. We chose one decision per screen with guided interaction patterns.

Transparency without technical overload

We considered hiding contract details entirely. That eroded trust. We chose abstracted interactions that still show what’s happening on-chain.

Playful branding, structured product

We considered pure meme aesthetics. That felt intimidating at product scale. We chose internet-native visuals with clean hierarchy and credibility.

RESULTS

Award-winning concept, proof that UX unlocks on-chain access

Developed and presented as an award-winning concept at BlockchainRio 2024 on LaChain, with a pump.fun-inspired model adapted for Brazilian Web3 audiences. The hackathon format forced ruthless prioritization: one decision per screen, transparent contract interactions, consumer-grade clarity.

Live demos validated that strong UX makes blockchain creation feel significantly more approachable. On-chain credibility stayed intact. Meme coin creation doesn’t need to feel like a developer tool.

REFLECTION

What I’d carry forward

Simplifying blockchain UX removes unnecessary friction, not depth. Users still need to see what happens on-chain. They just shouldn’t need a developer mindset to get there.
